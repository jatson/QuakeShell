QuakeShell — Design Directions

Explore visual approaches for the QuakeShell drop-down terminal. Each direction shows how the terminal integrates with the Windows desktop.

Direction 1: Minimal Pure

The v1 default — just terminal, full width, 30% height, 85% opacity. Zero chrome. The terminal IS the UI.

Key Elements

  • Full monitor width, 30% height
  • 85% opacity — desktop visible underneath
  • 2px accent blue bottom border
  • No tab bar, no title bar, no chrome
  • Tray icon only — no taskbar presence

When to Use

This is the core v1 experience. Single terminal session, maximum screen real estate for terminal content. The terminal appears as a native layer of the desktop, not a separate application.

Acrylic Effect

  • Win11 22H2+ only (graceful fallback to opacity)
  • Electron: win.setBackgroundMaterial('acrylic')
  • 70% opacity + blur — more readable than plain transparency
  • Desktop content blurred, not sharp — reduces distraction
  • Premium "native Windows" feel

Trade-offs

Acrylic adds GPU overhead and may impact performance on integrated GPUs. Only available on Windows 11 22H2+. Falls back to standard opacity on older systems. Could be a user-configurable option rather than default.

Theme System

  • xterm.js ITheme interface — full color control
  • All 16 ANSI colors + bg/fg + cursor + selection
  • Themes defined in JSON config
  • Hot-reloaded on config change
  • Community themes in Phase 2

Purpose

This direction demonstrates that the theme system is flexible enough to support radically different aesthetics while maintaining the same layout and interaction patterns. The retro theme proves theming works — users will create their own.

Resize Interaction

  • 6px drag handle at bottom edge of terminal
  • Cursor changes to ns-resize on hover
  • Handle highlights with accent color on hover
  • Small centered grip indicator (32px × 2px bar)
  • Drag up/down adjusts terminal height in real-time
  • Min height: 10% of screen, Max height: 90%

Auto-Save Behavior

  • New height saved to JSON config on drag-end
  • Debounced save — no writes during active drag
  • Persists across restarts and monitor changes
  • Height stored as percentage (adapts to resolution)
  • Per-monitor height in multi-monitor setups (future)
